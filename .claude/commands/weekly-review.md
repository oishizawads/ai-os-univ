# weekly-review

## Objective
1週間の活動を振り返り、週次レポートを作成する。

## When to Use
- 毎週金曜日（または週末）
- 週の区切りで進捗確認したいとき

## Process

### Step 1: 今週の活動を収集
以下を読む。
- 今週分の `daily_reports/YYYY-MM-DD.md`（月〜金）
- `SESSION_NOTES.md` の今週分エントリ
- `experiments/` 直近の `result.md`（コンペの場合）
- `meeting_notes/` の今週分（実務の場合）

### Step 2: 週次レポートを作成
`weekly_reports/YYYY-Www.md` に以下フォーマットで書く。

```markdown
# Weekly Report - {year}-W{week} ({MM/DD} - {MM/DD})

## Highlight
今週の最大の前進・成果を1〜2行で。

## Done
- 

## Carry Over
先週から持ち越したまま未完了のもの。
- 

## Risks / Blockers
- 

## Next Week Focus
1. 
2. 
3. 
```

### Step 3: STRATEGY.md / SESSION_NOTES.md を必要なら更新
重要な方針転換・知見があれば反映する。

## Hard Rules
- 「やった感」で終わらず、結果と次の問いを書く
- Carry Over は言い訳なく事実として残す
- Next Week Focus は最大3項目まで
