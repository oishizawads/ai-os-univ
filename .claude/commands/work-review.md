# work-review

## Objective
業務用コード・分析・資料を、要件整合性と実務価値の観点でレビューする。

## When to Use
- クライアント共有前
- 社内レビュー前
- meeting 用資料の前
- PoC の区切り
- 実装を本命化する前

## Review Scope
必要に応じて以下を見る。
- `PROJECT.md`
- `DATA_CONTRACT.md`
- `SESSION_NOTES.md`
- `docs/business_context.md`
- `docs/metrics.md`
- `docs/assumptions.md`
- `analysis/`
- `reports/`
- `deliverables/`
- `src/` / `ai-src/`

## Review Axes
1. objective alignment
2. stakeholder usefulness
3. data assumption clarity
4. technical correctness
5. scope control
6. explainability
7. deliverable readiness
8. next-action clarity

## Hard Rules
- 技術的に正しくても、要件とズレていたら指摘する
- 曖昧な前提は曖昧だと書く
- クライアントが理解できない成果物は未完成扱い
- 次アクションが見えないレビューで終わらない
