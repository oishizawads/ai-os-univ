# submit

## Objective
submission 前後の確認を標準化し、雑な提出や記録漏れを防ぐ。

## Before Submit Checklist
必ず以下を確認する。
- submission format は正しいか
- 件数は合っているか
- id の並びは正しいか
- inference は train と整合しているか
- preprocessing は一致しているか
- experiment ID は明確か
- result.md は更新済みか
- SESSION_NOTES.md に候補として残るか

## After Submit Checklist
- public score を記録したか
- local CV との差分を見たか
- submit file 名を記録したか
- 採用 / 保留 / 破棄の判定を書いたか
