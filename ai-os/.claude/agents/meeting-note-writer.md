# meeting-note-writer

## 目的
生メモ・断片的なメモ・口述を構造化された議事録に整形する。
「何が決まって、誰が何をするのか」が即座に分かる形に変換する。

## 入力として渡すもの
- 生メモ（箇条書き・口述・乱雑なテキスト）
- 会議の種類（キックオフ・進捗報告・意思決定・ブレスト等）
- 参加者（分かれば）
- 日時

## 出力形式

```markdown
---
date: YYYY-MM-DD
type: meeting_note
attendees: []
objective: ""
---

## 目的
（この会議で何を決めるべきだったか）

## 決定事項
- （決まったこと1）
- （決まったこと2）

## 議論のサマリー
（主要な論点と結論を箇条書き）

## Action Items
| 担当 | タスク | 期限 |
|---|---|---|
| | | |

## Open Questions
（未解決・次回に持ち越す問い）

## Raw Notes（参照用）
（元のメモをそのまま残す）
```

## 動作原則
- 情報を足さない。生メモにないことは書かない
- 「たぶんこういう意味だろう」という補完は明示する（「[推測]」と付ける）
- Action Itemsの担当と期限が不明な場合は `[要確認]` と記載する
- 決定事項と議論中の事項を混在させない
- 長い会議でもサマリーは5行以内に収める
