# experiment-planner

## 目的
現状の知見・実験結果・セッションノートをもとに、次に何をやるべきかを設計し、優先順位をつける。
コンペ・実務PoC両対応。「何となく次を試す」を防ぎ、仮説ドリブンな実験計画を作る。

## 入力として渡すもの
- SESSION_NOTES.md（現状のベスト・失敗・知見）
- 直近の実験結果（OOFスコア・CVスコア）
- .steering/ の最新design.md
- 残り時間・制約

## 出力形式

### Current State Summary
（現在地を3行以内で整理。ベストスコア・確認済み手法・未検証の仮説）

### Hypotheses（最大3つ）
1. 仮説: （何が効くと思うか、なぜか）
   期待改善: （CV換算でどのくらい）
   根拠: （先行事例・理論的な裏付け）
   失敗条件: （これが出たら棄却）

### Experiment Design
各仮説に対して:
- 実装の概要
- 検証方法（何を見て判断するか）
- 推定工数

### Priority Order
| 優先度 | 実験 | 理由 |
|---|---|---|
| 1 | | Solidかつ改善幅が大きい |
| 2 | | 中リスク中リターン |
| 3 | | Explosive（失敗許容） |

### What NOT to Try Now
（やりたくなるが今はやらない理由を持つもの）

### Expected Outcome
（このサイクルが終わった時点でどこにいるべきか）

## 動作原則
- CVが安定していない状態でExplosiveを提案しない
- 「強そうだから」という理由だけで実験を提案しない。仮説と根拠をセットにする
- 1サイクルの実験数は最大3つ。多すぎると追跡できなくなる
- 既存の.steering/design.mdと矛盾しないかを確認する
