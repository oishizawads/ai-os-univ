# Good Strategy Bad Strategy — 戦略フレームワーク

> 出典: リチャード・P・ルメルト『良い戦略、悪い戦略』（Good Strategy Bad Strategy: The Difference and Why It Matters）  
> 著者: Richard P. Rumelt（UCLA アンダーソン スクール 戦略論教授 / McKinsey 上級アドバイザー）  
> 用途: 戦略の設計・評価・意思決定。「良い戦略とは何か」の診断ツール

---

## 0. 根本原則

```
良い戦略には核心（カーネル）がある。
悪い戦略は目標・スローガン・願望を戦略と呼ぶ。
```

ルメルトの定義:
- **戦略とは**: 困難な状況に対処するための、診断・方針・行動の一貫した組み合わせ
- **戦略ではないもの**: 目標、ビジョン、スローガン、予算計画、意欲の表明

---

## Part 1: 戦略のカーネル（Kernel of Strategy）

良い戦略は必ずこの3要素を持つ。欠けていれば戦略とは言えない。

### 診断（Diagnosis）

```
状況の本質的な課題を特定し、複雑な現実を行動可能な問題として定義する
```

- 「何が本当の問題か」を明確にする
- 全ての問題を列挙するのではなく、**最も重要な課題の構造**を捉える
- 「強みが活きる文脈」を見つける
- 競合・市場・自社の相対的なポジションを診断する

**良い診断の特徴**:
- 複雑な状況をシンプルに要約できる
- 「だからこの行動が有効」という方向性が自然に生まれる
- 行動につながらない問題定義をしない

**悪い診断の例**:
```
× 「市場が厳しい環境にある」（自明で行動を生まない）
× 「競合が強くコストが高く顧客ニーズも変化している」（全部並べて構造化されていない）
○ 「顧客の意思決定が価格から体験に移行している転換期にある」（構造が見えている）
```

### 基本方針（Guiding Policy）

```
診断で特定した課題に、どのようにアプローチするかの方針
```

- **選択と集中**: 何をやるかより「何をやらないか」が重要
- 全員が賛同できる「無難な方針」は方針ではない（選択が伴わないから）
- 方針は「行動の方向性」を示すが、具体的行動そのものではない
- リソース配分の優先順位が暗示されている

**良い基本方針の特徴**:
- 特定の「賭け」が見える（これで勝負する、という方向性）
- 取捨選択が明確（AではなくBで戦う）
- 診断の課題に対して直接応答している

**例**:
```
診断: 「顧客の意思決定が価格から体験に移行している」
方針: 「価格競争から撤退し、体験の質で差別化するプレミアム戦略に集中する」
（価格競合には参加しない、という選択を含む）
```

### 一貫した行動（Coherent Actions）

```
基本方針を実行するための具体的行動のセット。各行動が互いに支え合う。
```

- 行動は方針を「実行」するものでなければならない
- **コヒーレンス**: 各行動が単独で動くより、組み合わさることで力が倍増する
- リソース配分（予算・人員・時間）の具体的決定を含む

**コヒーレンスの例（Apple の例 from 書籍）**:
```
方針: 「プレミアム体験で差別化する」

行動セット（コヒーレント）:
  ├─ ハード・ソフト・サービスを垂直統合する
  ├─ SKUを絞り込む（製品種類を少なくする）
  ├─ デザインに多大な投資をする
  ├─ 直営店で販売体験をコントロールする
  └─ 価格を下げない

（各行動が「プレミアム体験」という方針を支え、互いを強化している）
```

## Part 2: 悪い戦略の4パターン

ルメルトが「Bad Strategy」として定義する4類型。自分の戦略が当てはまっていないか確認する。

### 1. 空文（Fluff）

```
専門用語・大げさな言葉で中身のなさを隠す
```

- 実質的な内容を言っていないが、聞こえは良い
- 具体的な行動や選択に翻訳できない

**識別テスト**: 「で、具体的に何をするの？」に答えられるか

```
例:
× 「革新的なエコシステムを通じて、顧客価値を最大化するプラットフォーム戦略」
  → 何をするか、何をしないか、一切わからない

× 「デジタルトランスフォーメーションにより、競争優位を確立する」
  → 具体的な行動・選択・資源配分が一切ない
```

### 2. 課題に向き合わない（Failure to Face the Challenge）

```
本当に困難な選択を避け、全員が賛同する無難な内容にとどまる
```

- リスクや制約を直視しない
- 「誰も反対できない」戦略は、実は誰も選択していない
- 根本的な困難（例: 技術的劣位、資金不足、市場縮小）を脇に置く

**識別テスト**: 「この戦略のために、何を諦めたか？」に答えられるか

```
例:
× 「全ての事業部を均等に強化し、全市場でのプレゼンスを維持する」
  → 選択していない。全てをやろうとしている
```

### 3. 目標と戦略の混同（Goals Mistaken for Strategy）

```
「何を達成したいか（What）」を「どうやって達成するか（How）」と混同する
```

- 数値目標・ビジョン・意欲の表明を「戦略」と呼ぶ
- 達成方法・選択・資源配分が欠落している

**識別テスト**: 「どうやって？」に答えられるか

```
例:
× 「3年後に市場シェア30%を達成する」→ 目標。戦略ではない
× 「業界No.1のカスタマーサポートを提供する」→ 願望。どうやって？が抜けている
× 「イノベーションを推進し、成長を加速する」→ スローガン
```

### 4. 悪い戦略目標（Bad Strategic Objectives）

```
現場が何をすべきかわからない、または達成不能な抽象的目標
```

- 戦略目標は「行動に翻訳できる」ものでなければならない
- 「世界一」「最高品質」は検証不能
- 現場レベルで「これをやれば達成できる」というリンクがない

**識別テスト**: 現場のメンバーが「明日から何をすべきか」わかるか

## Part 3: 良い診断の作り方

### 診断のステップ

```
Step 1: 状況の事実を整理する（分析）
Step 2: 「最も重要な課題の構造」を1文で書く
Step 3: その構造から「行動の方向性」が自然に生まれるか確認する
Step 4: 強みが活きる文脈を見つける
```

### 転換点（Crux）の発見

ルメルトが「crux（核心）」と呼ぶ概念:
- **全ての問題の中で、解決すれば最も影響が大きい1点**
- 困難だが取り組めるレベルのもの
- 「ここを解けば他が連鎖的に解ける」ポイント

```
問い: 「このプロジェクト/課題において、
      これさえ解決すれば残りは比較的容易になる
      という1つの核心課題は何か？」
```

### 相対的優位性を見つける

良い診断は「自分が他より優れている点」を含む:

```
強みの問い:
  - 競合が簡単に真似できないものは何か？
  - 自分だけが持っているリソース・能力・ポジションは？
  - 競合が参入しにくい（コストが高い）領域はどこか？
```

## Part 4: コヒーレンス（一貫性）の確認

戦略の3要素が一本のストーリーでつながっているか。

| 要素 | 問い | NG例 |
|------|------|------|
| **診断** | 何が本当の課題か？ | 問題が複数あり構造化されていない |
| **方針** | その課題にどう向き合うか？ | 診断と関係ない方針 |
| **行動** | 具体的に何をするか？ | 方針に反する行動が含まれる |
| **コヒーレンス** | 行動は方針を実行しているか？ | 方針と無関係な行動がある |

### コヒーレンスチェック

```
診断: 「〇〇が本質的な課題だ」
方針: 「〇〇に対して、□□のアプローチで取り組む」
行動: 「具体的には△△・▲▲・◇◇を実施する」

チェック:
[ ] 方針は診断の課題に直接応答しているか？
[ ] 行動は方針を実現するためのものか？
[ ] 行動間に相乗効果・一貫性があるか？
[ ] 行動間に矛盾はないか？
```

## 実践チェックリスト

### 戦略を設計するとき

```
[ ] カーネルの3要素（診断・方針・行動）が揃っているか？
[ ] 診断: 状況の本質的課題が1文で書けるか？
[ ] 方針: 「何をやらないか」が明確か？
[ ] 行動: コヒーレントに方針を実行しているか？
```

### 戦略を評価するとき（悪い戦略チェック）

```
[ ] 「具体的に何をするの？」に答えられるか？（空文チェック）
[ ] 何かを諦める選択が含まれているか？（課題回避チェック）
[ ] 「どうやって達成するか」が書かれているか？（目標混同チェック）
[ ] 現場が明日から行動に移せるか？（目標品質チェック）
```

## Claudeに適用する場合のプロンプト

```
戦略のカーネルで考えてください。

1. 診断: この状況の本質的な課題を1文で特定する
2. 基本方針: その課題に対して何をするか・しないかを決める
3. 一貫した行動: 方針を実現するコヒーレントな行動セットを出す

悪い戦略チェック:
- 空文になっていないか（具体性があるか）
- 困難な選択を避けていないか
- 目標と混同していないか
---
[戦略的判断が必要な状況をここに記述]
```

## Anti-patterns

| パターン | 問題 |
|---------|------|
| **スローガンの量産** | 「革新」「最高」「No.1」を並べても戦略にならない |
| **全員賛成の戦略** | 選択がなければ戦略ではない。誰も反対しない＝何も選んでいない |
| **目標の羅列** | KPIをいくつも並べることと戦略は別物 |
| **コヒーレンスの欠如** | 行動が方針をバラバラな方向に向いている |
| **診断の省略** | 方針から始めると「なぜこの方針か」が語れない |
| **強みなき方針** | 自分の相対的優位性を活かしていない戦略 |
