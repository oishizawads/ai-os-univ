# ピラミッドプリンシプル — 思考フレームワーク

> 出典: バーバラ・ミント『考える技術・書く技術』（The Pyramid Principle: Logic in Writing and Thinking）  
> 著者: Barbara Minto（McKinsey & Company 初の女性コンサルタント）  
> 用途: 思考の構造化・文書作成・プレゼンテーション・あらゆるコミュニケーションの設計

---

## 0. 根本原則

```
人間は、ピラミッド状に構造化されたアイデアしか理解できない。
結論を最初に言え。理由は後だ。
```

ミントが発見した認知の法則:
- 人間の脳は「グループ化されたもの」しか記憶に残せない
- 読み手・聞き手は常に「So what?（だから何？）」と「Why so?（なぜそう言えるの？）」を心の中で問うている
- 書き手・話し手の仕事は、この問いを先回りして答えることである

---

## Part 1: SCQA フレームワーク（文書の冒頭設計）

あらゆる文書・プレゼンは**SCQA**で始まる。読み手を「問い」に引き込むための構造。

### S - Situation（状況）
- 読み手が**既に知っている・同意できる**背景事実
- 議論の起点となるコンテキスト
- 反論が起きないほど自明な事実から始める

### C - Complication（複雑化）
- Situationに**変化・問題・緊張関係**が生じた
- 「しかし〜」「ところが〜」「そこに〜が起きた」
- なぜ今この文書を読む必要があるか、の理由

### Q - Question（疑問）
- ComplicationによってSituationが揺らいだ結果、読み手が自然に持つ**問い**
- 文書全体が答えようとしている問い
- 多くの場合、明示しない（読み手が自然に感じるよう設計する）

### A - Answer（答え）
- 文書の**主メッセージ（トップ）**
- ピラミッドの頂点に来るもの
- QへのDirect Answerでなければならない

### SCQA の例

```
S: 「わが社の市場シェアは過去3年間、安定的に35%を維持してきた。」
C: 「しかし今年Q1、競合A社が新製品を投入し、シェアが5ポイント低下した。」
Q: （暗示）「どう対応すべきか？」
A: 「価格・品質・流通チャネルの3軸を同時に強化する反攻戦略を取るべきである。」
```

### SCQA の4つのパターン

| パターン | 形 | 使いどき |
|---------|-----|---------|
| **標準型** | S→C→Q→A | 最も一般的。聞き手が問題を認識している |
| **直入型** | S→A→C | 緊急性が高い・結論が既知の場合 |
| **懸念型** | C→Q→A | 問題が先行している場合 |
| **疑問型** | Q→A | 問いが明確なとき（「〇〇してください」への答え） |

## Part 2: ピラミッド構造

### 構造の原則

```
          [主メッセージ（結論）]
               ↙    ↓    ↘
        [根拠1]  [根拠2]  [根拠3]
         ↙ ↘    ↙ ↘    ↙ ↘
       [詳] [細] [詳] [細] [詳] [細]
```

- **縦の関係**: 上の主張を下が支える（Why so?）
- **横の関係**: 同じ階層の項目がMECEになっている
- **頂点の主張**: 1つの文に収まる明確なメッセージ

### 縦の論理（上下関係の2方向）

**So what?（下→上）: 要約・統合**
```
下位の複数ポイントを読み、「だから結局何が言えるか？」を導く
[事実A] + [事実B] + [事実C] → So what? → [主張]
```

**Why so?（上→下）: 支持・根拠**
```
上位の主張を読み、「なぜそう言えるか？」を確認する
[主張] → Why so? → [根拠1] [根拠2] [根拠3]
```

この2方向のテストを**全ての階層**で通過しなければならない。

### 横の論理（グループ化の2種類）

#### 帰納法的グループ化（Inductive）
- 同じ種類のものを並べ、共通の上位概念に統合する
- 「A、B、Cを実行すれば〇〇が達成される」

```
例:
上位: 「コスト削減には3つの手段がある」
  ├─ 根拠1: 「人件費を15%削減できる」
  ├─ 根拠2: 「調達コストを10%削減できる」
  └─ 根拠3: 「設備コストを8%削減できる」
```

- グループ内の項目は**同じ種類・同じ抽象レベル**でなければならない
- グループ内の順序: 構造的（時系列・重要度・分類）

#### 演繹法的グループ化（Deductive）
- 論理的な3段論法の連鎖
- **大前提 → 小前提 → 結論**の形

```
例:
大前提: 「コスト削減が急務の企業は、固定費を見直すべきだ」
小前提: 「わが社は今期コスト削減が急務の状況にある」
結論:  「わが社は固定費を見直すべきだ」
```

- 演繹法は主張の連鎖なので、第2の主張は第1の主張を受けなければならない
- 演繹法は読み手を「なるほど」と思わせるが、長いと重くなる

**実務では帰納法が主、演繹法は補完的に使う。**

## Part 3: キーライン（Key Line）

主メッセージ（頂点）を**直接支える第2レベルの主張群**。

### キーラインの条件
- それ自体が独立した主張である（事実・データではない）
- 主メッセージの「Why so?」に直接答える
- 全て合わせると主メッセージが成立する（MECE）

### キーラインの数
- **3〜5個が理想**
- 7個を超えたら再グループ化を検討
- 1個は「キーライン」ではなく「同語反復」になりがち

### キーラインの構造パターン

```
主メッセージ: 「〇〇社はX市場に参入すべきだ」

帰納型キーライン:
  ├─ 「X市場は成長している（機会）」
  ├─ 「〇〇社はX市場で競合優位を持つ（強み）」
  └─ 「参入コストはROIがプラスになる水準だ（実現可能性）」

演繹型キーライン:
  ├─ 「成長市場への参入は自社の戦略方針に沿っている」
  ├─ 「X市場は成長している」
  └─ 「よって参入は戦略方針に沿っている」
```

## Part 4: ストーリーラインの設計

### トップダウン型（Top-Down）
- 結論の仮説を先に持ち、それを支える根拠を積み上げる
- 問いに対する答えが既にある場合
- **実務で推奨** — ピラミッドを頂点から設計する

```
Step 1: 主メッセージ（答え）を決める
Step 2: キーラインを考える（Why so?）
Step 3: 各キーラインを支える詳細を考える
Step 4: SCQAで冒頭を設計する
```

### ボトムアップ型（Bottom-Up）
- 事実・データを積み上げて結論に到達する
- 探索的・分析的な場面
- ロジックを確認する際にも使う

```
Step 1: 言いたいことを全て書き出す
Step 2: 似ているものをグループ化する
Step 3: グループを1つの主張に統合する（So what?）
Step 4: 主張をさらにグループ化して主メッセージを導く
```

## Part 5: MECEの切り方

ピラミッドの横の論理はMECEが基本。MECEの主な切り方：

| 切り方 | 例 | 使いどき |
|-------|----|---------|
| **プロセス分解** | 入力→処理→出力 | 流れに沿って分析 |
| **構造分解** | 部門別・地域別・製品別 | 組織・カテゴリの分析 |
| **数式分解** | 売上 = 単価 × 数量 | 計算可能な指標 |
| **対比** | 内部/外部、短期/長期 | 2分法 |
| **フレームワーク** | 3C、4P、SWOT | 既存の枠組みが有効な場合 |
| **仮説列挙** | 原因A/B/C | 原因が複数考えられる場合 |

## 実践チェックリスト

### 文書・プレゼン設計前

```
[ ] SCQAが設計されているか（S・C・Q・Aを書けるか）
[ ] 主メッセージが1文で書けるか
[ ] その主メッセージはQへの直接の答えか
```

### 構造チェック

```
[ ] 全ての階層でWhy so?（下が上を支えているか）が成立するか
[ ] 全ての階層でSo what?（上が下を統合しているか）が成立するか
[ ] 同じ階層の項目はMECEか（重複なし・漏れなし）
[ ] キーラインは3〜5個か
```

### 文章化前

```
[ ] 結論が最初に来ているか（結論ファースト）
[ ] 各段落の先頭文が段落全体の要約になっているか
[ ] 「接続詞」が論理の繋がりを正確に示しているか
```

## Anti-patterns

| パターン | 問題 |
|---------|------|
| **結論が最後** | 「〇〇で△△で××で、したがって〜」→ 読み手は結論まで待てない |
| **So whatがない** | 事実を並べるだけ。読み手が「で？」となる |
| **ピラミッドの混在** | 同じ階層に主張と事実が混在している |
| **キーラインが多すぎる** | 8個以上→ グループ化できていない証拠 |
| **SCQAのCがない** | なぜ今読む必要があるかが伝わらない |
| **MECEの崩れ** | 重複があると同じことを2回言っている。漏れがあると反論される |
