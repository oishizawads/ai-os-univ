# ロジカルシンキング — 思考フレームワーク

> 出典: 照屋華子・岡田恵子『ロジカル・シンキング』（東洋経済新報社）  
> 補足: ロジックツリー・MECE実践は McKinsey & Company の手法を参照  
> 用途: 論理的に考え・論理的に伝えるための基盤。ピラミッドプリンシプルの実践的補完

---

## 0. 根本原則

```
論理的であるとは、「So what?」と「Why so?」に常に答えられる状態を維持することである。
```

照屋・岡田が定義するロジカルシンキングの2本柱:
1. **論理的に考える**: 物事の構造を正確に把握し、問題の本質を見極める
2. **論理的に伝える**: 相手が「So what?」「Why so?」と感じないよう、先回りして答える

---

## Part 1: 2つのキー問答

あらゆるコミュニケーションで意識すべき2つの問いと答え。

### So what?（だから何？）

- **使う向き**: 事実・情報・分析結果 → 主張・結論
- **問い**: 「これらの事実から、何が言えるのか？」
- **役割**: 個別の情報を意味のある洞察に変換する

```
失敗例:
  「売上が10%下落しました。競合が新製品を出しました。顧客アンケートで不満が増えています。」
  → 「So what?」が答えられていない

成功例:
  「これらの事実から、わが社の製品競争力が低下していることがわかります。（So what）」
  「なぜなら、売上下落・競合参入・顧客不満という3つの指標が同時に悪化しているからです。（Why so）」
```

### Why so?（なぜそう言えるの？）

- **使う向き**: 主張・結論 → 根拠・事実
- **問い**: 「その主張は何によって支えられているか？」
- **役割**: 主張の信頼性・説得力を担保する

```
失敗例:
  「この戦略は成功します。」（根拠なし）

成功例:
  「この戦略は成功します。（主張）
   なぜなら、①市場成長率が年15%、②競合の空白領域が存在する、
   ③自社の強みが直接活かせる、という3点が根拠です。（Why so）」
```

## Part 2: 論証の3要素（Toulminモデル）

主張を論理的に組み立てる基本構造。

```
[根拠（Grounds）] ──→ [主張（Claim）]
         ↑
   [論拠（Warrant）]
```

| 要素 | 定義 | 例 |
|------|------|---|
| **主張（Claim）** | 最も伝えたいこと | 「この施策を今すぐ実施すべきだ」 |
| **根拠（Grounds）** | 主張を支える事実・データ | 「顧客流出率が月3%に達している」 |
| **論拠（Warrant）** | 根拠から主張へのつながりを正当化するルール | 「流出率3%超は緊急対応の閾値を超えている」 |

### 論拠が欠けている例（論理の飛躍）

```
根拠: 「競合がAI機能を実装した」
主張: 「わが社もAI機能を実装すべきだ」
欠けている論拠: 「競合と同じ機能を持つことが差別化に必要である」
              （本当にそうか？という問いが生じる）
```

## Part 3: MECE（Mutually Exclusive, Collectively Exhaustive）

### MECEとは

- **Mutually Exclusive（相互排他）**: 重複がない（同じことを2回言わない）
- **Collectively Exhaustive（網羅的）**: 漏れがない（重要な観点を落とさない）

MECEが崩れると:
- **重複があると**: 同じ根拠を重複してカウントし、主張が過大になる
- **漏れがあると**: 重要な反証・観点が抜け落ち、主張が脆くなる

### MECEの切り方（7パターン）

**① プロセス分解**
```
「製品が売れるまで」= 認知 → 興味 → 検討 → 購買 → リピート
```

**② 構造分解**
```
「売上」= 商品A + 商品B + 商品C（足し合わせると全体になる）
```

**③ 数式分解（最も強力）**
```
売上 = 客数 × 客単価
客数 = 新規顧客 + 既存顧客
客単価 = 購入点数 × 平均単価
```
→ ツリーを作れるため、数値ベースの分析に直結

**④ 対比・2分法**
```
内部 / 外部
短期 / 長期
コスト / 収益
コントロール可能 / 不可能
```

**⑤ 時系列分解**
```
過去（原因分析）/ 現在（現状把握）/ 未来（施策）
```

**⑥ フレームワーク活用**
```
3C: Customer（顧客）/ Competitor（競合）/ Company（自社）
4P: Product / Price / Place / Promotion
SWOT: 強み / 弱み / 機会 / 脅威
バリューチェーン: 調達→製造→販売→サービス
```

**⑦ 仮説列挙**
```
「売上が落ちた原因」= 製品品質の問題 / 価格の問題 / 営業力の問題 / 外部環境の変化
（これらが重複せず、全てを合わせると原因の全体をカバーする）
```

### MECEチェック

```
【重複チェック】
→ 2つの項目を合体させたとき、意味が変わらないなら重複している

【漏れチェック】
→ 「他には？」を3回問う
→ 全項目を足し合わせたとき「全体」になるか確認する
```

## Part 4: ロジックツリー

MECEを使って問いを構造的に分解するツール。

### ツリーの3種類

**① Why ツリー（原因分析）**
```
目的: 「なぜ問題が起きているか」の原因を探る
方向: 問題 → 原因1 → 原因1-1 / 原因1-2
          → 原因2 → ...
問い: 「なぜ〇〇が起きているか？」を繰り返す

例:
「売上が落ちている」
  ├─ 「新規顧客が減っている」
  │    ├─ 「認知度が下がっている」
  │    └─ 「競合の広告が増えている」
  └─ 「既存顧客の購買頻度が下がっている」
       ├─ 「製品満足度が下がっている」
       └─ 「代替品が出てきた」
```

**② How ツリー（解決策）**
```
目的: 「どうすれば解決するか」の手段を展開する
方向: 目標 → 手段1 → 手段1-1 / 手段1-2
          → 手段2 → ...
問い: 「どうすれば〇〇を達成できるか？」を繰り返す

例:
「売上を30%上げる」
  ├─ 「客数を増やす」
  │    ├─ 「広告費を増やす」
  │    └─ 「紹介プログラムを作る」
  └─ 「客単価を上げる」
       ├─ 「高単価商品を作る」
       └─ 「クロスセルを強化する」
```

**③ What ツリー（構造分解）**
```
目的: 対象物の構成要素を明確にする
方向: 全体 → 構成要素1 → ...
問い: 「〇〇はどのような要素から成るか？」

例:
「顧客体験」
  ├─ 「購入前体験」（認知・情報収集）
  ├─ 「購入時体験」（接客・手続き）
  └─ 「購入後体験」（配送・サポート・リピート）
```

### ツリーを作る手順

```
Step 1: 頂点の問いを一文で定義する
Step 2: 最初の分岐をMECEで切る（フレームワーク or 仮説 or 構造）
Step 3: 各ブランチをさらにMECEで分解する
Step 4: 「これ以上分解しても有益な示唆が出ない」レベルまで掘る
Step 5: Why so? / So what? で縦の論理を確認する
```

## Part 5: 論理の2パターン（再掲・詳細）

### 演繹法（Deduction）

```
大前提（一般的なルール）
    + 小前提（特定の事実）
    ↓
    結論

例:
大前提: 「コスト削減が急務の企業はコア事業に集中すべきだ」
小前提: 「わが社は今、コスト削減が急務の状況にある」
結論:   「わが社はコア事業に集中すべきだ」
```

注意: 大前提が間違っていると結論も間違う。大前提の妥当性を問え。

### 帰納法（Induction）

```
事実A + 事実B + 事実C + ...
         ↓
     共通パターン
         ↓
      一般化（主張）

例:
「顧客Aは価格ではなくサポートで判断した」
「顧客Bは価格ではなくサポートで判断した」
「顧客Cは価格ではなくサポートで判断した」
→「この市場では、価格よりサポートが購買決定を左右している（仮説）」
```

注意: 帰納法は反例が1つあれば崩れる。サンプルの代表性を問え。

## 実践チェックリスト

### 考えるとき

```
[ ] So what?（この情報から何が言えるか）を答えられるか
[ ] Why so?（その主張の根拠は何か）を答えられるか
[ ] 論拠（根拠と主張のつながりのルール）は明確か
[ ] ロジックツリーはMECEになっているか
```

### 伝えるとき

```
[ ] 主張・根拠・論拠の3点セットが揃っているか
[ ] 聞き手が「So what?」と言いたくなる瞬間はないか
[ ] 聞き手が「Why so?」と言いたくなる瞬間はないか
[ ] 演繹か帰納か、論理の種類を意識しているか
```

## Anti-patterns

| パターン | 問題 |
|---------|------|
| **事実の羅列** | So whatがなく、聞き手が意味を自分で作らなければならない |
| **根拠なき主張** | Why soに答えられない。説得力ゼロ |
| **論拠の欠落** | 根拠と主張の間に飛躍がある |
| **MECEの崩れ（重複）** | 同じ原因を別の名前で2回カウントしている |
| **MECEの崩れ（漏れ）** | 重要な反証・視点が抜けており、主張が一面的 |
| **ツリーが深いだけ** | 深く掘っているが、So whatに繋がっていない |
| **フレームワークの盲信** | 3Cを使ったが、このケースに適切かどうかを考えていない |
