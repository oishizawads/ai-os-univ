## Summary
不確実な状況での意思決定は「情報を集めてから決める」ではなく「決められる状態を作ってから集める」。
可逆性とインパクトで意思決定のコストを見積もる。
情報収集は仮説を棄却するためにやる。

## Core Principles
- 意思決定の遅延もコスト（機会損失・依存タスクのブロック）
- 全情報が揃ってから決めるのは「決めない」という選択
- 可逆な決定は早く決める。不可逆な決定は慎重に
- 「正しい答え」より「棄却できない仮説の中でベストな選択」を目指す

## 分類フレームワーク

### 可逆性 × インパクト
| | 低インパクト | 高インパクト |
|---|---|---|
| 可逆 | **即決**（考えすぎない） | **仮実行**（小さく試す） |
| 不可逆 | **プロセス簡略化**して決める | **慎重に**（仮説・リスク明文化） |

### 情報収集の打ち切り判断
- 追加情報で決定が変わるか？ → 変わらないなら今すぐ決める
- 収集コスト vs 意思決定の質向上 → 逓減するポイントで止める
- 締め切りがあるか → あるなら「現時点でのベスト」で動く

## Decision Rules
- まず「この決定は可逆か」を問う
- 可逆 → 最低限の情報で決めて実行し、結果で更新
- 不可逆 → 前提・リスク・棄却条件を書いてから決める
- 「もっと調べてから」が2回続いたら、実は調査が目的になっている

## 意思決定ログの書き方
```
決定内容: （1文）
前提: （この決定が成立する条件）
却下した選択肢: （と却下理由）
主なリスク: （と軽減策）
棄却条件: （これが起きたら見直す）
```

## Anti-patterns
- HIPPO（役職が高い人の意見で決まる）
- 全員合意を目指す（意思決定の責任が曖昧になる）
- 「データが揃ったら」と言い続ける
- 決定後に新情報が来るたびに判断を変える（方針がブレる）
- リスクを書かずに楽観的シナリオだけで判断する

## Eval
- 決定の根拠が1文で言えるか
- 可逆性を確認したか
- 却下した選択肢と理由を書いたか
- 棄却条件（何が起きたら見直すか）を定義したか
