## Phase別タスク詳細

### Phase 1 ｜ 今すぐ：既存データでPoC

```
できること：
  ・地域 × サービス × 月次の手配難リスクマップ
  ・供給過多エリアの特定（営業強化の根拠）
  ・社内合意を取るための「見える化」

限界：
  ・業者の実力が見えないため精度に上限あり
  ・「誰をアサインすべきか」はわからない
```

### Phase 2 ｜ データ基盤整備後：精度が上がる

```
やること：
  ・マスタの一元管理
  ・フラット構造の正規化・ETL高速化
  ・業者ケイパビリティ情報の収集（評価加点インセンティブ）

追加でできること：
  ・業者の実質処理能力を加味したスコアリング
  ・「この地域は3社いるが実質1社分の供給力」が見える
  ・基幹システムへの依存度が下がり、カラム追加コストが激減
```

### Phase 3 ｜ トランザクションログ活用後：競合優位へ

```
やること：
  ・作業開始〜完了のリードタイム記録
  ・業者ごとの実績（到着時間・解決率・クレーム率）の蓄積
  ・通話ログの構造化

できること：
  ・「電話が来た瞬間に誰を送るか」のリアルタイム最適化
  ・業者がデータを入れると仕事が来る仕組み（入力インセンティブの自動化）
  ・フィードバックループによる継続的な精度改善
```
